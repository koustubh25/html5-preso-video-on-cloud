Need to deliver Video over Multiple Devices

According to the statistics obtained from an online statistics portal

  • Video Traffic will occupy about 79% of the total Internet Traffic by 2018.
  • There will be about 25 billion devices connecting to the Internet by 2020.

    • So, there is a need to deliver Live Video over Multiple Devices.

Difficulties with Streaming across Devices

  • Different Device - Different codec/Format
  • Different Device - Different Streaming Protocols - e.g. Apple HLS, MPEG-DASH, Microsoft Smooth Streaming
  • Different Device - Different Processing capabilities e.g. frame rate, bit rate, etc.

Resource Provisioning

Challenges in Resource Provisioning

  • The cloud servers have diverse capacities(e.g. cpu, memory etc.) and lease prices (e.g. US$).
  • The lease duration is not infinitesimally short (e.g., 1 hour for Amazon EC2).
  • The configuration and boot-up time can take from 10-30 minutes for Amazon’s EC2 instance [1].
  • There is heterogeneous distribution of cloud servers all over the world.
    • Therefore, the servers cannot be simply started and terminated at any given time.

Conclusion

  • We proposed a Framework to deliver Live Video over Multiple Platforms by carrying out Video Encoding in a Cloud Environment.
  • We developed and verified a Neural Network based Prediction Model to predict User Demand for Live Video that fits well into our proposed Framework [4].
  • We formulated the resource provisioning algorithm as a multi-objective Integer Linear Programming problem and performed simulations.

Future Work

  • Comparison of Performance with similar implementations e.g.any commercially available Prediction Services(e.g. Google Prediction API).
  • Introduce a technique to select from the various Pareto optimal solutions based on user priority.
  • Introduce other constraints in the optimization problem e.g. QoE.
